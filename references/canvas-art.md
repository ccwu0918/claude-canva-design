# 靜態藝術創作模式（Canvas Art）

> 整併自 canvas-design skill。適用於純藝術輸出場景：海報、靜態資訊圖、PNG/PDF 藝術品。
> 不需要互動性、品牌識別度需求弱、輸出交給設計師或印刷時，優先走此模式。

---

## 何時走這個模式（vs 走 HTML 主流程）

| 場景 | 走哪個 |
|------|--------|
| 海報 / 藝術品 / 無品牌靜態視覺 | **本模式（Canvas Art）→ PNG/PDF** |
| 品牌有明確 Logo + 色值的靜態資訊圖 | HTML 主流程 → 截圖 PNG |
| 需要互動 / 幻燈片 / App 原型 | HTML 主流程 |
| 動畫 → MP4/GIF | HTML 主流程 + video-export |

**判斷原則**：需要「被感受的藝術品」→ 本模式；需要「被使用的設計物」→ HTML 主流程。

---

## 完整兩步流程

### Step 1 · 設計哲學創作（輸出 .md）

在動手前，先建立一個**視覺哲學**（不是版面配置或模板）——它會透過以下方式被詮釋：
- 形式、空間、色彩、構圖
- 影象、圖形、形狀、圖案
- 極少文字（僅作視覺點綴）

#### 核心理解

- **收到的是**：使用者的一些微妙輸入或說明——作為基礎，不應限制創作自由
- **創作的是**：一個設計哲學 / 美學運動
- **接下來做的**：將這個哲學**用視覺表達出來**——製作 90% 視覺設計、10% 必要文字的作品

把這個過程想成：先寫一個藝術運動的宣言，再製作這件藝術品。

#### 哲學構成要素

**命名運動**（1-2 字）：「Brutalist Joy」/ 「Chromatic Silence」/ 「Metabolist Dreams」

**闡述哲學**（4-6 段，簡潔但完整）——涵蓋視覺本質如何透過以下方式呈現：
- 空間與形式
- 色彩與材質
- 尺度與節奏
- 構圖與平衡
- 視覺層級

**撰寫守則**：
- **避免重複**：每個設計面向只提一次
- **反覆強調工藝**：哲學必須多次強調最終作品應看起來像花了無數小時精心製作——「meticulous craftsmanship」「product of deep expertise」「painstaking attention」「master-level execution」
- **留下創作空間**：對美學方向足夠具體，但夠簡練讓視覺詮釋有餘地

#### 哲學範例（精簡版，實際寫 4-6 段）

**「Concrete Poetry」**
透過宏大的形式和粗獷的幾何表達。巨大色塊、雕塑感排版（超大單字+微小標籤）、Brutalist 空間分割。用視覺重量和空間張力表達理念，而非文字解釋。

**「Chromatic Language」**
色彩作為主要資訊系統。幾何精確性讓色域創造意義。排版極簡——小型無襯線標籤讓色彩場域溝通。Josef Albers 的色彩互動 meets 資料視覺化。

**「Analog Meditation」**
透過紋理與留白靜默沉思。紙質紋理、墨水暈染、大量留白。攝影和插畫主導。排版輕聲細語（小而剋制）。日本攝影書美學。

**「Organic Systems」**
自然聚落和模組化生長圖案。圓潤形式、有機排列。透過視覺圖解、空間關係、圖示系統傳達資訊。

**「Geometric Silence」**
純粹秩序與剋制。網格精確性、大膽攝影或鮮明圖形、戲劇性留白。瑞士形式主義 meets Brutalist 材質誠實。

哲學輸出為 `.md` 檔案。

### Step 2 · 捕捉微妙參照（關鍵步驟）

在創作 Canvas 前，辨識原始請求中的**微妙概念線索**。

主題是**嵌入藝術中的微妙、利基參照**——不總是字面的，永遠精緻。熟悉該主題的人應該直覺感受到，而其他人只是欣賞一幅精彩的抽象構圖。像爵士樂手引用另一首曲子——只有懂的人才能聽出來，但所有人都欣賞這段音樂。

---

### Step 3 · Canvas 創作（輸出 .pdf 或 .png）

確立哲學和概念框架後，在 canvas 上表達它。

**核心原則**：
- 以設計哲學為基礎
- 創作單頁、高視覺性、設計主導的 PDF 或 PNG（除非被要求更多頁）
- 通常使用重複圖案和完美形狀
- 加入稀疏、臨床感排版和系統性參照標記
- 用簡單短語或細節低調錨定作品
- 使用有意圖性和凝聚感的有限色板
- 擁抱用分析性視覺語言表達人類體驗的悖論

**文書處理**：文字永遠極少且視覺優先。大多數情況字型應纖細。所有字型使用必須設計感強，優先視覺溝通。沒有元素超出頁面邊界，沒有重疊。所有元素必須在 canvas 邊界內，留適當邊距。

**工藝要求（非協商）**：為達到人工製作品質（非 AI 生成感），創作看起來像花了無數小時的作品。讓它看起來像一位頂尖領域的人以痛苦的細心勞作每個細節。確保構圖、間距、色彩選擇、排版——一切都散發出專家級工藝。

輸出為單一可下載的 .pdf 或 .png，連同使用的設計哲學 .md。

---

### Step 4 · 精煉 Pass（必做）

**假設使用者說了「還不夠完美，必須是一件傑作，彷彿即將在美術館展出。」**

精煉時，**避免新增更多圖形**；而是精煉已有的，讓它極為銳利。不是加新元素，而是問：「如何讓已有的東西更像藝術品？」

回到程式碼，進一步精煉/拋光，製作成哲學設計的傑作。

---

### 多頁選項

要求額外頁面時，沿相同設計哲學創作更多具創意但明顯不同的頁面。將它們打包在同一 .pdf 或多個 .png。把第一頁視為整本等待填滿的咖啡桌書中的單頁。讓它們以非常有品位的方式幾乎講述一個故事。
